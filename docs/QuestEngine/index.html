Quest.js hands off data to Quest-Data. The command is run from a Sabre-Enabled Server. The system executes an appropriate response given the %Chance.

quest.js

This command requires Permission Level 1, Hyper Level 1, and 1 Quest Key.
The system is loaded. Specific developers (Permission Level 4) are allowed access to the force argument which forces the system's chance calculator to obtain a user-defined integer between 0-100.
Usage Example: quest force 96

In a typical setup, the user is greeted with the header data.
HLVL: Hyperlevel
HQKY: Quest Keys
HDTK: Dark Tickets
followed by a randomly generated room based on your %Chance score.
